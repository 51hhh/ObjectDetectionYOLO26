# 仓库文件夹架构规范

## 1. 目标

为了便于训练、推理、版本管理和后续部署，仓库目录应按“配置、脚本、文档、数据、输出”进行分层。

---

## 2. 推荐目录结构

```text
ObjectDetectionYOLO26/
├─ configs/                     # 配置文件
│  └─ train_volleyball.yaml
├─ docs/                        # 文档
│  ├─ 01_训练环境搭建与训练步骤.md
│  ├─ 02_部署推理文档.md
│  └─ 03_仓库文件夹架构规范.md
├─ scripts/                     # 可执行脚本
│  ├─ prepare_dataset.py
│  ├─ train_volleyball.py
│  └─ infer_volleyball.py
├─ coco/                        # 训练数据集目录
│  ├─ annotations/
│  ├─ images/
│  ├─ labels/
│  ├─ data.yaml
│  └─ summary.json
├─ runs/                        # 训练输出与推理输出
├─ images.zip                   # 原始正样本压缩包
├─ negative_samples.zip         # 原始负样本压缩包
├─ yolo26n.pt                   # 预训练模型
├─ .gitignore
└─ README.md
```

---

## 3. 各目录职责

### configs/
存放训练和推理相关配置。

### docs/
存放项目说明、环境搭建文档、训练步骤、部署指南。

### scripts/
存放所有可执行脚本。
要求：
- 不把脚本散落在根目录
- 脚本内部统一通过项目根路径拼接资源文件

### coco/
存放整理后的训练数据。
可视为“中间产物 + 训练输入数据”。
如果数据量较大，不建议纳入 Git。

### runs/
存放训练日志、权重、推理结果。
属于输出目录，通常不应提交到 Git。

---

## 4. Git 管理建议

建议提交到 Git 的内容：
- `configs/`
- `docs/`
- `scripts/`
- `.gitignore`
- `README.md`

通常不建议提交到 Git 的内容：
- `.venv/`
- `runs/`
- `coco/images/`
- `coco/labels/`
- 大体积压缩包
- 中间缓存

---

## 5. 命名规范建议

### 文档命名
建议使用序号前缀，保证阅读顺序明确：
- `01_训练环境搭建与训练步骤.md`
- `02_部署推理文档.md`
- `03_仓库文件夹架构规范.md`

### 脚本命名
建议使用动词或任务名：
- `prepare_dataset.py`
- `train_volleyball.py`
- `infer_volleyball.py`

### 输出目录命名
建议按任务语义命名：
- `runs/volleyball_train`
- `runs/volleyball_predict`

---

## 6. 后续扩展建议

如果后续加入更多模型或任务，可扩展为：

```text
configs/
  train_volleyball.yaml
  train_xxx.yaml
scripts/
  train_volleyball.py
  infer_volleyball.py
  export_onnx.py
models/
  best.pt
```

如果后续要做服务化部署，可新增：
- `service/`：接口服务代码
- `deploy/`：部署脚本与配置
- `tests/`：测试代码
