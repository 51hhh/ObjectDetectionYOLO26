# 部署推理文档

## 1. 当前部署目标

当前仓库部署分为两部分：

1. Python 基线推理
   - 使用 `scripts/infer_volleyball.py`
   - 直接加载 `weight/best.pt`
2. AGX + ZED 实时部署
   - 使用 `scripts/export_onnx.py` 导出 ONNX
   - 在 `deploy/agx_zed/` 下构建 C++ / TensorRT / ZED 运行时
   - 使用 ZED 双目深度，但只取 `LEFT` 图像做检测
   - 基于 `DEPTH + 内参` 解算球相对相机坐标 `(x, y, d)`
   - 预留球从相机坐标转换到世界坐标的接口
   - 支持持续运行和 OpenCV 可视化窗口

## 3. Python 基线推理

推理脚本：
- `scripts/infer_volleyball.py`

示例：

```bash
python ./scripts/infer_volleyball.py "你的图片或视频路径"
```

常用参数：

```bash
--weights   模型权重路径
--imgsz     推理尺寸，默认 640
--conf      置信度阈值，默认 0.25
--iou       NMS IoU 阈值，默认 0.15
--device    推理设备，如 0 或 cpu
--name      输出目录名
--save-txt  保存 txt 格式结果
--save-conf 保存置信度到 txt
--show      打开可视化窗口
```

---

## 4. ONNX 导出

导出脚本：
- `scripts/export_onnx.py`

示例：

```bash
python ./scripts/export_onnx.py --weights ./weight/best.pt --imgsz 640
```

默认输出：
- `deploy/agx_zed/models/best.onnx`

可选参数：

```bash
--weights       输入 .pt 权重
--output        输出 ONNX 路径
--imgsz         导出输入尺寸，默认 640
--opset         ONNX opset，默认 17
--dynamic       启用动态尺寸
--simplify      启用 simplify
```

说明：
- 第一版建议使用静态 `640x640`
- 第一版建议先走 FP16 静态链路，不直接上 INT8
- YOLO26 的 ONNX 输出 shape 必须以实际导出结果为准，不能照搬 `example` 的 `[1,5,N]` 假设

## 5. AGX + ZED 运行时结构

部署工程路径：
- `deploy/agx_zed/`

当前运行时包含：
- `camera/zed_camera.*`
  - ZED `LEFT` 图像与 `DEPTH` 深度采集
  - 读取相机内参 `fx, fy, cx, cy`
- `detector/trt_engine.*`
  - TensorRT engine 加载与推理
- `detector/postprocess.*`
  - YOLO26 end2end `(1,300,6)` 输出解码
- `geometry/depth_sampler.*`
  - 深度窗口采样
- `geometry/projection.*`
  - 像素 + 深度 + 内参 -> 相机点 `P_cam`
- `geometry/world_transform.*`
  - `P_cam -> P_world` 接口预留
- `pipeline/ball_pipeline.*`
  - 采集、检测、深度、投影串联入口
- `common/visualization.*`
  - 状态文字、检测框、坐标 overlay
- `main.cpp`
  - 持续循环、日志输出、GUI / headless 控制

### 当前坐标定义

对外相机相对坐标：
- `x`：左右，右正左负，单位米
- `y`：上下，上正下负，单位米
- `d`：到相机的欧氏距离，单位米
- `z`：相机前向深度，单位米

内部保留：
- `P_cam = (Xc, Yc, Zc)`
  - `Xc`：相机右向
  - `Yc`：相机上向
  - `Zc`：相机前向深度

投影公式：

```text
Xc = (u - cx) / fx * Zc
Yc = - (v - cy) / fy * Zc
Zc = depth_z

d  = sqrt(Xc^2 + Yc^2 + Zc^2)
```

其中：
- `(u, v)` 为检测框中心像素
- `depth_z` 为该点邻域深度窗口的中位数

## 6. AGX 运行前提

建议在 AGX 上准备好以下环境：
- JetPack / CUDA / TensorRT
- ZED SDK
- OpenCV 4
- `yaml-cpp`
- 可访问的 ZED X 相机
- 已准备好的 `best.engine`

当前配置文件：
- 主配置：`configs/deploy/agx_zed_yolo26.yaml`
- 相机位姿：`configs/deploy/camera_pose.yaml`

当前已验证有效的 ZED 模式：
- `HD1200@60`

### Engine 获取方式

仓库运行时直接加载现成 engine，不在 C++ 启动时自动从 ONNX 构建。

推荐流程：

1. 在本仓库导出 ONNX

```bash
python ./scripts/export_onnx.py --weights ./weight/best.pt --imgsz 640
```

2. 在目标 AGX 上用 TensorRT 工具链生成 engine

```bash
trtexec --onnx=./deploy/agx_zed/models/best.onnx --saveEngine=./deploy/agx_zed/models/best.engine --fp16
```

说明：
- engine 最好在目标 AGX 上生成，避免 TensorRT 版本不匹配
- 当前仓库已忽略 `*.onnx` 和 `*.engine`，默认不提交模型构建产物

## 8. AGX 启动方法

建议始终显式传入配置文件路径，不依赖工作目录。

### 8.1 Headless 模式

把 `configs/deploy/agx_zed_yolo26.yaml` 中的 `show` 改为 `false`：

```yaml
show: false
```

启动：

```bash
./deploy/agx_zed/build/agx_zed_runtime ./configs/deploy/agx_zed_yolo26.yaml ./configs/deploy/camera_pose.yaml
```

行为：
- 不打开任何窗口
- 持续运行
- 控制台输出 `No detection`、`Detection but invalid depth` 或带坐标的 `Tracking ball`
- 适合 SSH / 无桌面环境

### 8.2 GUI 模式

把 `show` 改为 `true`：

```yaml
show: true
```

启动：

```bash
./deploy/agx_zed/build/agx_zed_runtime ./configs/deploy/agx_zed_yolo26.yaml ./configs/deploy/camera_pose.yaml
```

窗口内容：
- ZED `LEFT` 实时画面
- 检测框和中心点
- `score`、`class`
- `x / y / d / z`
- 当前状态文本
- `Press q or ESC to exit`

退出方式：
- 按 `q`
- 按 `ESC`

### 8.3 GUI 可用前提

`show: true` 只表示程序尝试创建 OpenCV 窗口，不代表当前会话一定具备显示能力。

常见可用场景：
- AGX 本地桌面终端
- 已正确转发显示并具备有效 `DISPLAY` 的图形会话

常见不可用场景：
- 普通 SSH 终端直接启动
- 没有桌面登录的纯无头环境

如果你是远程 SSH 到 AGX，优先先用 `show: false` 验证检测链路；需要窗口时，再切到 AGX 本地桌面会话启动。

## 9. 常见故障排查

### 9.1 `engine file not found`
- 检查 `configs/deploy/agx_zed_yolo26.yaml` 中的 `engine` 路径
- 确认 `deploy/agx_zed/models/best.engine` 已存在于 AGX

### 9.2 `deserializeCudaEngine failed`
- engine 与当前 TensorRT 环境不兼容
- 在当前 AGX 上重新执行 `trtexec`

### 9.3 `ZED grab failed`
- 检查相机连接、供电、SDK 与权限
- 确认当前配置的 `resolution` / `fps` 组合有效
- 当前已验证 `HD1200@60`

### 9.4 `show=true` 但没有窗口
- 这是显示环境问题，不是模型或深度逻辑本身的问题
- 请改为在 AGX 本地桌面终端启动
- 或先切回 `show: false` 做链路验证

### 9.5 持续输出 `No detection`
- 球没有进入画面，或模型当前没有检出
- 先确认检测框是否偶尔出现
- 可以先用 Python 基线脚本对同场景图片做静态对齐

### 9.6 持续输出 `Detection but invalid depth`
- 表示检测已命中，但检测框中心邻域缺少有效深度
- 检查球是否过远、过近、反光或被遮挡
- 检查 `depth_window_radius`、`min_depth_m`、`max_depth_m`
